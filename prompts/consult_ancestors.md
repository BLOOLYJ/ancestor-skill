# 翻阅族谱 📜

> 仪式第三步：审查历史，以史为镜，避免重蹈覆辙

## 目标

以史为镜，可以知兴替。在动手之前，翻翻「族谱」——看看前人在相关代码上做过什么、踩过什么坑、留过什么话。

## 执行步骤

### 1. 查看相关文件的 Git 历史

```bash
git log --oneline -20 -- {相关文件路径}
```

关注：
- 谁最近改过这些文件？
- 改动频率如何？（频繁修改 = 不稳定区域）
- 最近的 commit message 是 fix/hotfix 吗？（= 有历史 bug）

### 2. 查看整体最近变更

```bash
git log --oneline -10
```

关注：
- 有没有正在进行的大型重构？
- 有没有刚上线的功能可能有冲突？

### 3. 检查牌位族谱（如果有）

读取 `chronicles.md`，寻找相关记录：
- 是否有类似的变更曾经出过问题？
- 是否有前人留下的警告？
- 是否有相关的技术债记录？

### 4. 检查牌位祖训（如果有）

读取 `teachings.md`，检查：
- 是否有相关的编码规范？
- 是否有红线禁区会被触及？
- 是否有推荐的做法可以参考？

## 输出格式

```
📜 族 谱 已 阅 ，前 人 之 鉴 📜

👣 前人足迹：
   - {最近相关 commit 摘要}
   - {文件修改频率}

⚠️ 前车之鉴：
   - {相关历史问题}
   - {类似变更的教训}

📋 悬案未决：
   - {相关技术债}
   - {未解决的相关问题}

📖 祖训提示：
   - {相关的最佳实践}
   - {需要遵守的规范}

前人栽树后人乘凉，前人踩坑后人绕行。
```

如果没有发现相关历史：

```
📜 族 谱 已 阅 📜

此处尚无前人足迹，你将成为第一个。
开荒不易，更要小心谨慎。

继续三叩九拜……
```

## 要点

- 频繁被 hotfix 的代码区域是高风险区域
- 如果同一个文件在短时间内被多人修改，要特别注意冲突
- 前人留下的 TODO、FIXME、HACK 注释也是重要线索
- 族谱查阅不是走过场，真的要从历史中发现潜在风险
