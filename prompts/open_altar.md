# 开坛设位 🏮

> 仪式第一步：加载项目上下文，确保 AI 理解代码库全貌

## 目标

在开始任何分析或执行之前，先「开坛」——即加载当前项目的完整上下文。就像祭祀前要先把祖宗的牌位摆好、供品备齐一样。

## 执行步骤

### 1. 查找项目牌位

检查 `${SKILL_DIR}/tablets/` 下是否有当前项目的牌位。匹配方式：
- 精确匹配 slug
- 用户直接指定
- 根据当前目录名推断

### 2. 加载牌位内容（如果有）

读取以下文件：
- `meta.json`：项目元信息（名称、技术栈、历史数据）
- `teachings.md`：祖训（最佳实践、编码规范、红线禁区）
- `chronicles.md`：族谱（历史教训、已知 bug、重大事件）

### 3. 扫描当前项目状态

即使有牌位，也要扫描当前状态：
```bash
git status
git log --oneline -5
```

### 4. 宣告开坛

**有牌位时**：
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🏮 坛 已 开 ，位 已 设 🏮

📋 项目：{project_name}
🛠️ 技术栈：{tech_stack}
📅 立牌日期：{created_at}
⚠️ 历史教训：{chronicles_count} 条
📖 祖训条目：{teachings_count} 条
🔥 累计保佑：{blessing_count} 次

香火已点燃，请告知所求之事……
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**无牌位时**：
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🏮 坛 已 开 ，但 尚 未 立 位 🏮

未找到此项目的牌位。
祖宗虽然万能，但也得知道保佑的是谁。

输入 /create-tablet 为当前项目立一块牌位
或直接说出所求之事，祖宗先凑合着保佑
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 要点

- 开坛是所有后续步骤的基础，必须先执行
- 牌位信息将在后续步骤中作为检查依据
- 如果项目没有牌位，仪式仍然可以继续，只是缺少历史上下文
- 语气要庄重但不失幽默，像一个经验丰富的老祖宗
