# 回向功德 🙏

> 仪式第七步：经验沉淀，更新牌位，福荫后人

## 目标

每一次祭祀都不应白费。回向功德 = 经验沉淀 + 牌位更新。今天踩的坑记到族谱里，今天学到的好方法写进祖训里，让后面的 session、后面的人都能受益。

## 执行步骤

### 1. 总结本次经验

回顾整个仪式过程，提炼值得记录的内容：

**新增祖训**（如果有）：
- 本次发现了什么好的做法？
- 有什么值得固化为规范的？
- 有什么容易被忽略但很重要的检查点？

**新增族谱**（如果有）：
- 本次踩了什么坑？
- 有什么差点出问题的地方？
- 有什么值得后人警醒的事？

### 2. 更新牌位（如果项目有牌位）

**更新 meta.json**：
```json
{
  "blessing_count": "+1",
  "last_blessing": "{当前ISO时间}",
  "updated_at": "{当前ISO时间}"
}
```

**追加 teachings.md**（如果有新祖训）：
用 `Edit` 工具在文件末尾追加：
```markdown
## {日期} 新增

- {新的祖训条目}
```

**追加 chronicles.md**（如果有新教训）：
用 `Edit` 工具在「重大事件」节追加：
```markdown
- {日期}：{事件描述}
```

### 3. 收尾

```
🙏 功 德 回 向 ，福 荫 后 人 🙏

本次祭祀圆满完成。

📝 经验沉淀：
   - {新增的祖训（如有）}
   - {新增的族谱（如有）}

🏮 牌位状态：{已更新 / 无牌位，未更新}
🔥 香火延续：累计第 {n} 次保佑

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
愿代码无 bug，
愿上线不翻车，
愿祖宗保佑。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

如果本次没有值得记录的新经验：

```
🙏 功 德 回 向 🙏

本次祭祀圆满完成。
此次一切顺利，无新坑可记。
祖宗甚慰。

🔥 香火延续：累计第 {n} 次保佑

愿代码无 bug，愿上线不翻车，愿祖宗保佑。
```

## 要点

- 回向是仪式的收尾，但也是下一次仪式的起点
- 经验沉淀要具体，不要写空洞的总结
- 不是每次都必须有新增祖训/族谱，如果真的没有就不要硬凑
- 牌位的价值在于长期积累，每一次小小的记录都很有意义
- 最后的祝语是固定的，给仪式一个仪式感的结尾
