# 焚香沐浴 🕯️

> 仪式第二步：透彻分析需求，拆解任务，明确目标和约束

## 目标

焚香以静心，沐浴以净身。在动手之前，先把要做的事情想清楚、说明白。这一步是需求分析和任务拆解。

## 分析维度

### 1. 所求何事（What）

用一句话精准概括核心需求。不多不少，抓住本质。
- 是新功能还是 bug 修复？
- 是重构还是优化？
- 是技术方案选型还是架构设计？

### 2. 涉及范围（Where）

列出这个变更会影响到的范围：
- 哪些文件需要修改？
- 哪些模块会被影响？
- 是否涉及跨服务调用？
- 是否涉及数据库变更？

### 3. 约束条件（Constraints）

明确限制条件：
- 时间约束（是否紧急？有没有 deadline？）
- 技术约束（框架限制？版本要求？）
- 兼容约束（是否需要向后兼容？）
- 性能约束（有没有 SLA 要求？）

### 4. 利益相关（Who）

谁会被这个变更影响？
- 用户（功能变化？体验变化？）
- 其他开发者（接口变化？依赖变化？）
- 运维（部署变化？配置变化？）
- 业务（数据影响？流程影响？）

## 输出格式

```
🕯️ 焚 香 沐 浴 ，静 心 明 意 🕯️

📌 所求何事：{一句话概括核心需求}
📂 涉及范围：
   - {文件/模块 1}
   - {文件/模块 2}
   - ...
⚖️ 约束条件：{关键约束}
👥 利益相关：{影响方}

香已燃尽，心已明，翻阅族谱以鉴今……
```

## 要点

- 这一步的目的是「慢下来」，避免 AI 不经思考就开始执行
- 如果需求不清晰，应该在这一步就提出来，而不是带着疑问继续
- 约束条件往往是容易忽略的，要主动去想
- 分析完毕后自然过渡到下一步「翻阅族谱」
