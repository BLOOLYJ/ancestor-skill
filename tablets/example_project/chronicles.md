## 已知的坑

### 💀 float 精度问题（2026-02-10）

**症状**：用户充值 100 元，上了 33 柱 3 元的香后，余额显示 0.9999999999999858 而不是 1.00
**原因**：金额字段使用了 `float` 类型
**修复**：全部改为 `Decimal`
**教训**：涉及钱的字段永远不要用浮点数

### 💀 状态机绕过（2026-02-25）

**症状**：部分供品显示「已供奉」但实际没有扣除用户余额
**原因**：有人直接用 SQL 改了 `offerings` 表的 `status` 字段，绕过了状态机
**修复**：数据库层面加了触发器校验
**教训**：状态变更必须走状态机，这是铁律

### 💀 N+1 查询（2026-03-05）

**症状**：「我的供品」页面加载需要 8 秒
**原因**：循环查询每个供品的详情，100 个供品 = 101 次数据库查询
**修复**：改为 `selectinload` 预加载
**教训**：列表页面必须检查是否有 N+1 查询

### ⚠️ 未解决：Legacy 祈福格式解析器

`legacy_blessing_handler.py` 处理 17 种历史格式，代码 800 行无注释无测试。目前能跑，但没人敢动。需要找时间重构 + 补测试。
