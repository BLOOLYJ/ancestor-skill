# 祖宗.skill 产品需求文档（PRD）

## 产品概述

**祖宗.skill** 是一个 Cursor / Claude Code Agent Skill，通过将「祭祀仪式」映射为系统化的工程检查流程，提升 AI 辅助开发的决策质量。

### 核心价值主张

1. **强制慢思考**：通过仪式化的步骤，强制 AI 在执行前完成全面的分析和检查
2. **系统化检查**：三叩九拜覆盖正确性、健壮性、可维护性三大维度，九项具体检查
3. **经验沉淀**：牌位系统让项目知识持久化，跨 session 复用
4. **风险可视化**：卦象系统将模糊的"可能有风险"转化为量化的判断

### 目标用户

- 使用 Cursor / Claude Code 进行日常开发的工程师
- 希望提升 AI 辅助编码质量的团队
- 喜欢有趣工具的开发者

## 用户流程

### 流程一：完整祭祀

```
触发 → 开坛设位 → 焚香沐浴 → 翻阅族谱 → 三叩九拜 → 卜卦问吉 → 祖宗显灵 → 回向功德
```

适用场景：重要变更、架构设计、上线前检查、技术方案选型

### 流程二：快速上香

```
触发 → 速读上下文 → 三拜速查 → 速出卦象 → 执行 + 速签
```

适用场景：日常小改、Bug 修复、参数校验等低风险操作

### 流程三：创建牌位

```
触发 → 信息录入 → 项目扫描 → 生成牌位文件
```

适用场景：首次为项目建立知识档案

## 数据结构

### 牌位（Tablet）

每个项目的牌位存储在 `tablets/{slug}/` 目录下：

#### meta.json

```json
{
  "name": "项目名称",
  "slug": "url-friendly-slug",
  "created_at": "ISO 8601",
  "updated_at": "ISO 8601",
  "tech_stack": ["技术1", "技术2"],
  "description": "项目描述",
  "blessing_count": 0,
  "last_blessing": null,
  "risk_level": "unknown|low|medium|high"
}
```

#### teachings.md（祖训）

包含：
- 技术规范
- 代码风格
- 红线禁区
- 推荐做法
- 按日期追加的新条目

#### chronicles.md（族谱）

包含：
- 项目纪年（大事记）
- 已知的坑（含症状、原因、修复、教训）
- 重大事件
- 前人遗言

## 仪式步骤映射

| 仪式步骤 | Prompt 文件 | 工程实践 | 输入 | 输出 |
|---------|-------------|---------|------|------|
| 开坛设位 | `open_altar.md` | 项目上下文加载 | 牌位文件 | 上下文摘要 |
| 焚香沐浴 | `burn_incense.md` | 需求分析 | 用户描述 | 结构化需求 |
| 翻阅族谱 | `consult_ancestors.md` | 历史审查 | git log + 族谱 | 历史风险 |
| 三叩九拜 | `three_kowtows.md` | 质量检查 | 方案 + 代码 | 9 项评估 |
| 卜卦问吉 | `divination.md` | 风险评估 | 9 项评估结果 | 卦象 + 卦辞 |
| 祖宗显灵 | `blessing.md` | 执行任务 | 全部上下文 | 实际输出 |
| 回向功德 | `dedication.md` | 经验沉淀 | 执行结果 | 更新牌位 |

## 卦象体系

六级卦象，基于九拜量化结果：

| 等级 | 符号 | ✅ 数量 | ❌ 数量 | 建议行动 |
|------|------|--------|--------|---------|
| 大吉 | 🟢 | 9 | 0 | 放心执行 |
| 吉 | 🔵 | 7-8 | 0 | 留意注意项 |
| 小吉 | 🟡 | 5-6 | 0 | 谨慎执行 |
| 平 | 🟠 | - | 1 或 3+⚠️ | 建议先处理 |
| 凶 | 🔴 | - | 2 | 暂停 |
| 大凶 | ⚫ | - | 3+ | 重新设计 |
